# 动作序列使用说明

本文档说明如何设定角色按顺序执行一系列动作。

---

## 一、使用方式

### 方式 1：视频模式操作栏（推荐）

1. 进入 **视频模式**（Camera Mode）
2. 在底部操作栏点击 **播放** 图标（▶）
3. 在下拉菜单中选择预设序列：
   - **开场介绍**：招手 → 数字1指 → 展示
   - **讲解列举**：数字1 → 2 → 3
   - **指向系列**：Pointing → PointingForward → ReachingOut
   - **开心互动**：开心 → 招手 → 展示

4. 序列播放完成后，角色会自动回到待机状态

### 方式 2：代码调用

在任意组件中通过 `viewer.model?.playMotionSequence` 调用：

```tsx
import { useGlobalStore } from '@/store/global';
import { MOTION_SEQUENCES } from '@/config/motionSequences';

// 在组件内
const viewer = useGlobalStore((s) => s.viewer);

// 播放预设序列
viewer.model?.playMotionSequence([...MOTION_SEQUENCES.intro], () => {
  console.log('序列播放完成');
  viewer.model?.loadIdleAnimation(); // 回到待机
});
```

### 方式 3：自定义序列

```tsx
import { MotionFileType } from '@/libs/emoteController/type';
import type { MotionSequenceItem } from '@/libs/emoteController/type';

const mySequence: MotionSequenceItem[] = [
  { type: 'preset', preset: 'female_greeting' },
  { type: 'preset', preset: 'number_meter_1' },
  { type: 'url', fileType: MotionFileType.FBX, url: '/animations/Pointing.fbx' },
];

viewer.model?.playMotionSequence(mySequence, onComplete);
```

---

## 二、序列项格式

每个动作可以是以下两种之一：

| 类型 | 格式 | 示例 |
|------|------|------|
| **预设** | `{ type: 'preset', preset: string }` | `{ type: 'preset', preset: 'female_greeting' }` |
| **URL** | `{ type: 'url', fileType, url }` | `{ type: 'url', fileType: MotionFileType.FBX, url: '/animations/Pointing.fbx' }` |

### 预设名称（preset）

参考 `docs/角色动作与表情参考.md`，常用预设包括：

- `female_greeting` - 招手
- `female_appeal` - 展示
- `number_meter_1` ~ `number_meter_5` - 数字手势
- `female_happy` - 开心
- `female_stand` - 站立待机
- `idle` - 空闲

---

## 三、添加自定义序列

在 `src/config/motionSequences.ts` 中新增：

```ts
export const MOTION_SEQUENCES = {
  // ... 现有序列
  myCustom: [
    { type: 'preset', preset: 'female_greeting' },
    { type: 'url', fileType: MotionFileType.FBX, url: '/animations/MyMotion.fbx' },
  ] as MotionSequenceItem[],
};
```

并在 `PlaySequence.tsx` 的 `SEQUENCE_LABELS` 中添加对应标签。

---

## 四、注意事项

1. **播放顺序**：动作按数组顺序依次播放，每个动作播放一次（不循环）
2. **中断**：切换动作、停止说话等会中断当前序列
3. **完成回调**：`onComplete` 在全部播放完成后调用
4. **待机**：建议在 `onComplete` 中调用 `loadIdleAnimation()` 让角色回到待机
